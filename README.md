# SpringKnowledgeCourseWork

**SpringKnowledgeCourseWork** — учебный проект-курсовая, в котором реализована простая, но показательная **веб-система базы знаний**.  
Цель проекта — показать полный путь: от доменной модели и БД до REST-API и SPA-фронтенда на Vue.

---

## Задача проекта

Проект имитирует небольшую «внутреннюю» базу знаний: набор статей, разделённых по категориям, с тегами, авторами и версиями.  
Это типичный кейс для корпоративных знаний, документации или учебных материалов.

Через этот пример демонстрируется:

- как спроектировать доменную модель (статьи, категории, теги, авторы, комментарии, версии и т.д.);
- как описать её с помощью JPA и связать с реляционной БД;
- как открыть к этой модели REST-API на Spring Boot;
- как построить одностраничный интерфейс на Vue, который работает поверх этого API.

---

## Основной функционал

В общем виде приложение умеет:

- **Работать со статьями базы знаний**  
  - создание новой статьи;
  - просмотр подробной страницы статьи;
  - редактирование содержимого;
  - удаление при необходимости.

- **Организовывать контент**
  - привязка статей к **категориям**;
  - использование **тегов** для более гибкого поиска и группировки;
  - хранение информации об **авторе** статьи.

- **Хранить метаданные**
  - дата создания;
  - дата последнего обновления;
  - связи с версиями и/или комментариями (если реализовано в конкретных сущностях).

Фактически это «скелет» системы документации/knowledge-base, который легко дорастить под свои задачи.

---

## Архитектура

Проект разделён на два независимых модуля:

- `backend/` — серверная часть на **Java + Spring Boot**  
  Отвечает за:
  - REST-эндпоинты для работы с сущностями;
  - бизнес-логику;
  - доступ к базе данных через Spring Data JPA.

- `frontend/` — клиентская часть на **Vue.js**  
  Отвечает за:
  - отображение списка статей и отдельных страниц;
  - формы создания/редактирования;
  - запросы к backend по HTTP (REST API).

Такое разделение повторяет типовую архитектуру современного веб-приложения: отдельный API-сервер и SPA-клиент.

---

## Технологический стек

### Backend

- **Java**  
- **Spring Boot**  
- **Spring Web** (реализация REST-контроллеров)  
- **Spring Data JPA** (работа с БД через ORM)  
- Реляционная СУБД (настраивается в конфигурации: URL, логин, пароль)

### Frontend

- **Vue.js**  
- **JavaScript**  
- Сборка и запуск через `npm`/`yarn` (скрипты описаны в `frontend/package.json`)

## Что показывает этот проект

На уровне курсовой/учебного примера этот репозиторий демонстрирует:

- умение строить **доменную модель** и связи между сущностями;
- работу с **JPA / ORM** и реляционной базой;
- создание **REST-слоя** на Spring Boot;
- интеграцию backend + frontend в одно работающее приложение;
- базовые навыки работы с **SPA на Vue**.

## Как можно использовать проект дальше

- как основу для собственной **системы базы знаний**;
- как **шаблон CRUD-приложения** на стеке Spring Boot + Vue;
- как пример к курсовой/учебной работе по веб-разработке:
  - «Разработка веб-приложения базы знаний»,
  - «Система управления статьями»,
  - или любой похожей теме.

Проект специально сделан максимально «приземлённым» и понятным, чтобы его было легко развивать: можно добавить поиск, пагинацию, авторизацию, роли пользователей и более сложную работу с версиями статей.
